📦 B4 客户运营工具箱

休食组 · AM 角度梳理「从找标的 → 看市场 → 写内容」全流程可用的工具/SKILL/数据资产
面向：电商业务三部 · 行业 AM 更新：2026-06-02 维护：大门（朱锦程） 建议浏览器：Chrome/Safari
这是个什么文档？ 把"运营一个 B4 客户"日常会用到的工具/skill/看板/数据集，按 4 步运营动作串起来；每一步告诉你 用什么 → 怎么调 → 输入啥 → 出来啥 → 注意啥。 目标：AM 拿到一个商家时，从工具箱顶部往下扫一遍就知道该做什么。
Step 1

🎯 标的筛选

从名下盘子里，挑出值得这双月重点投精力的商家。

Step 2

🔍 买点 + 内容趋势

定位商家在站内的市场机会，找到值得讲什么、对谁讲。

Step 3

🆕 新品上市洞察

商家有新品要打，给它一份 SPU 级的内容机会清单。

Step 4

✍️ 笔记内容提质

商家已经在发的笔记，判断哪些是好内容，怎么改更好。

STEP 1

🎯 标的筛选 — 这双月，AM 重点投谁？

解决：盘子太大、不知道把时间投在哪个商家
核心问题：B3/B4 商家上百个，单看 GMV 增幅会被偶发投放骗（"一场达人投放 ≠ 可持续"）。 需要从 底盘 · 动能 · 场域 · 意愿 多维度找出"给资源能跳 B5+"的真潜力股。

线上看板 → 打开看板

把休食组 964 个 B3/B4 商家按 V2 公式自动分级，S 档优先看（~50 家），A 档排期跟进（~144 家），B 档自动化触达。

看板入口
https://zhudamen528-code.github.io/dgate-board-v2/b3b4-screening/?v=v2
更新频率
每周一 08:00 自动刷新（cron 调度）
怎么用
① 进看板看 S 档清单 → ② 按"店铺名 + AM"筛自己名下 → ③ 用 1.2 文档看模型解释 → ④ 把 S 档商家拉进双月 OKR
典型问题
"这双月我该重点投谁？"、"我名下有哪些黑马值得抢救？"、"老板要看我的重点商家清单"

⚠️ 注意：S 档≠承诺一定跳档，是"信号集中"，最终决策仍要看 AM 对商家底盘、节奏的判断。GMV ≥40 万的临界商家模型预测力较弱（看板里有 ⚠️ 标记）。

1.2 📘 V2 模型设计原理 + 6 月回测报告

REDoc 文档 → 打开文档

不是看板的"使用说明"，是"为什么这么打分 + 真实跳档率验证"。AM 用来跟商家/老板解释"我为啥说这家是 S 档"。

核心结论
S 档跳档率 29.2% vs 对照组 4.0%（+629%）— 模型有真预测力
看哪几页
① 公式：组合分 = 0.4×场域约束 + 0.6×动能
② 4 维子分独立展示（不出黑箱总分）
③ 局限性（罚单/连续2月增长字段缺失）
典型场景
商家问"我有什么信号被你看到了"、老板问"模型是不是瞎打的"

📌 配套用法：跟商家沟通时打开 1.1 看板的"信号详情"，对照 1.2 文档解释每一维分数。

STEP 2

🔍 买点洞察 + 内容趋势 — 商家该讲什么、对谁讲

解决：商家说"我不知道发什么内容"，AM 缺乏数据化建议
核心问题：电商商家做内容种草，最大痛点不是"不会拍"而是"不知道卖什么点"。 这一步用灵犀的数据能力，从 痛点（品类/产品/成分/人群） 四个维度反推商家该讲什么。

2.1 🎯 灵犀-找买点（find-buy-point）

灵光 Skill → 安装/触发

输入"行业 / 品牌 / SPU / 人群包"任一组合，AI 自动跑灵犀多维数据交叉分析，输出 S/A/B 分级的潜力买点卡 + 动态 PPT。免去"自己下数 + 人工提炼买点"的两三天工作。

触发词
找买点、买点洞察、挖掘买点、买点分析
调用方式
在 DIBP 对话框直接说自然语言，可指定行业 / 品牌 / SPU / 人群包
耗时
约 15-30 分钟
输入示例 ①（行业级）
我想看滋养食补行业近30天的买点洞察

📄 滋养食补行业买点洞察 (REDoc)

🎞 动态 PPT

输入示例 ②（品牌级）
分析 SOILETA 品牌的潜力买点

📄 SOILETA 品牌买点洞察 (REDoc)

🎞 动态 PPT（12页）

💡 给 AM 的用法：跟商家月会前一晚跑一次，会议上直接用动态 PPT 过买点，比让商家自己看灵犀洞察后台高效 10 倍。

2.2 🧩 灵犀手动洞察 · 4 大痛点视角

灵犀后台 → 方法论原文

Skill 跑不出来或想做更精细分析时，按"4 大痛点"维度手动到灵犀后台对应模块看。每个视角对应不同决策场景。

① 品类痛点
挖本品类 / 竞争品类痛点 → 灵犀【洞察 → 行业 → 搜索洞察】看搜索热词 / 飙升词 / 下游词
e.g. 亚麻籽油 → "能热炒吗"、"和其他油区别"、"不会吃"
② 产品痛点
挖本品牌 / 产品痛点 → 灵犀【洞察 → 趋势分析 → 品牌/SPU】+【度量 → 品牌口碑】
e.g. SOILETA → "品牌信任度"、"产品口味"
③ 成分痛点
挖核心成分痛点 → 灵犀【洞察 → 行业 → 笔记洞察】看热门笔记词云
e.g. omega-3 → "促宝宝脑发育"、"抗炎减脂"、"延缓衰老"
④ 人群痛点
已知目标人群 → 灵犀【洞察 → 人群 → 我的人群 + 洞察任务】
e.g. 减脂人群 × 饮料冲调 → "补蛋白"、"稳食欲"、"排钠补钾"

⚠️ 权限注意：灵犀部分类目/品牌需 OA 申请，禁止申请全行业。权限申请指南

2.3 📡 趋势洞察生成器（xhs-trend-insights）

灵光 Skill → 安装/触发

买点是"商家应该讲什么"，趋势洞察是"这一周/这一个月平台在火什么"。两者结合 → 找到「商家买点 × 平台热点」的交集，做最容易爆的内容。

触发词
趋势洞察 / 热点雷达 / 场景深挖 / 内容机会
典型输入
「La Mer 油皮水有什么内容机会」（SPU 级 ⭐推荐）
「帮我做一个完整的热点洞察」（品牌级）
「热点雷达」（仅扫热点，30 分钟）
耗时
50-75 分钟（含 P0/P1/P2 热点分级 + 3 层深挖 + SPU 嫁接策略）
输出
HTML 报告：热点雷达 + 场景深挖 + 6 行选题建议表

💡 适合周度巡检：每周一跑一次"热点雷达"，5 个组员共享一份报告，节省 5 倍时间。

STEP 3

🆕 新品上市洞察 — 商家有新品要打，给一份机会清单

解决：商家新品 SKU 要上线，AM 缺一份系统化的内容/竞品/人群参考
核心问题：商家要发新品时，"参考竞品成功路径 + 找出差异化卖点 + 锁定目标人群" 是 AM 经常被问的三件事，手动做要 1-2 天，Skill 化后 30 分钟出报告。

3.1 🎁 新品上市洞察（new-product-launch-insight）

灵光 Skill → 安装/触发

输入新品信息（品牌 + 新口味/新规格），自动从站内同品类爆款里学路径：投放节奏、卖点结构、目标人群、竞品打法 → 一份 PPT/REDoc 报告。

典型场景
① 商家月初告知"我们 6 月要上 XX 新口味"
② 商家做新品类切入（如休食品牌要做谷物棒）
③ 老品牌焕新（包装/概念升级）
实战 case
奥利奥 - 开心果乳酪 / 海盐椰椰新口味
六月鲜 - 白松露新品
输出
HTML 报告 / REDoc 文档 / 可视化 PPT

📌 对比 2.1 找买点：2.1 看的是"商家已有 SPU 的潜力买点"；3.1 看的是"还没上的新品该怎么打"。两者互补。

3.2 🔬 新品开发洞察（brandnew-product-insight）

2T Agent Skill → 安装/触发

在"商家还没决定开什么新品"时用 — 输入品牌 + 类目，AI 反推用户在小红书的真实需求空白，给出开品方向建议。

触发方式
Step 1：先输入 2t-brandnew-product-insight（建议执行，稳定触发）
Step 2：「跑一下 蒙牛、冰激凌 的新品开发洞察报告」
使用时机
商家年初/季度初做新品规划阶段（早于 3.1 上市阶段）

💡 适合跟商家"产品 + 运营"联席会前跑一次，把"用户想要什么"摆到桌面上。

3.3 💎 闭环电商宝藏潜力品发掘器（potential-category-finder）

SKILL（已升级自动取数 v15） → 安装/触发

从"平台已有单品"里挖被低估的潜力 SKU — 对商家"已经卖但没火"的品做潜力评估，找出该重点投流/做内容的 SPU。

触发示例
「帮我做一次潜力品类挖掘，附件是滋补赛道 3&4 双月的 SPU GMV 底表」（旧版需上传底表）
新版 v15 已实现自动取数
SOP 文档
v15 SOP（一页讲清楚）
STEP 4

✍️ 笔记内容提质 — 商家在发的笔记，好不好？怎么改？

解决：商家月发上百篇笔记，但不知道哪些是"好内容"，AM 也没标准说清
核心问题：商家发笔记最大的浪费是"过软（产品提不到）+ 过硬（生硬植入）"。 用平台官方的「三感六度」标准来批量打标，让"什么是好内容"有客观尺度。

4.1 🏷️ 三感六度 · 好内容打标（good-content-tagger）

灵光 Skill（营销实验室出品） → 安装/触发

输入一批笔记 ID（最多 30 篇/次），自动取 SQL 取数（标题/正文/ASR/OCR/类目/SPU）→ 按规则打标 good / ok / bad → REDoc 文档返回。

触发词
「好内容打标」「三感六度打标」「沙隆好内容」「好内容分析」
前置 Skill
需先安装 sql-development
输入
笔记 ID 列表（换行/逗号分隔，≤30 篇）+ 触发词
输出字段
笔记分数（good/ok/bad）/ 评分理由 / for_k 要点 / 卖转买结果 / 打标时间
输入示例
69bb7e4f000000001b023ad2
69bfad4d0000000021007fba
698c5106000000001503b25c
给我进行沙隆好内容打标
评分规则速记

BAD：过软/过硬/夸大杜撰/低俗

OK：未命中 BAD + 七大维度命中 ≤3

GOOD：未命中 BAD + 七大维度命中 ≥4

七大维度：受众精准 / 兴趣激发 / 场景融入 / 情绪共鸣 / 需求满足 / 封面标题自然呈现 / 正文自然呈现

💡 给 AM 的用法：① 拿商家近 30 天 Top 笔记跑一遍，找出 GOOD 案例做 SOP 沉淀
② 拿 BAD 笔记给商家，对照"过软/过硬"做明确的改稿建议
📚 详细规则：三感六度使用介绍

4.2 🔬 内容诊断与机会分析（content-diagnosis）

灵光 Skill → 安装/触发

不是单篇打标，是品牌/SPU 级别的内容资产体检 — 跟竞品对比 → 看本品/竞品的题材覆盖、买点布局，找"人有我无 / 我有人优 / 持续深耕" 3 类机会点。

触发词
内容诊断 / 笔记诊断 / 内容分析
输入
「帮我做 [品牌] 的内容诊断」「对 [SPU 名称] 做内容资产诊断」
3 步 SOP
Step 1 竞品对比 → Step 2 人群偏好 → Step 3 机会发现

📌 对比 4.1：4.1 是"看单篇好不好"；4.2 是"整盘内容布局对不对"。建议商家月会前先跑 4.2 看大盘缺口，再用 4.1 改具体笔记。

4.3 💡 社区热点 × SPU 内容灵感碰撞

灵光 Skill → 安装/触发

商家"知道要发但没素材想"的时候用 — 基于灵犀内容/话题广场 + SPU 数据，结合最近社区热点，给具体的创作切角灵感。

触发词
内容灵感 / 灵感报告 / 小红书内容策略 / 种草内容 / 软广笔记
输入示例
使用 xhs-content-inspiration 技能，产出【XX SPU / 赛道】灵感洞察报告
输出
默认网页；可指定输出 PPT / 文档
